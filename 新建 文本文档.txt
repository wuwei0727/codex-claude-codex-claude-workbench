## 2. 项目目标

构建一个 Windows 优先的 Codex / Claude 日常工作台。

核心目标：

- 管理 Codex / Claude Provider。
- 支持官方登录配置和 API Relay 配置共存。
- 支持一键切换 Provider。
- 支持配置自动备份。
- 支持 Codex 会话搜索、导出、删除。
- 支持 MCP / Skills / Prompts 管理。
- 将 Codex++ 注入能力作为可选独立模块，不能影响核心功能。
- 第一版先服务个人日常使用，不追求支持 Gemini / OpenCode / 云同步 / 多语言等扩展能力。

## 3. 技术栈

已确定技术栈：

- 框架：Tauri 2
- 前端：React + TypeScript + Vite
- UI：shadcn/ui 风格组件 + TailwindCSS
- 数据请求：TanStack Query
- 后端：Rust
- 数据库：SQLite
- 配置管理：TOML / JSON / SQLite
- 安装包：Windows NSIS 或 MSI

## 4. 设计原则

- Windows 优先，界面走 Windows 11 / Fluent 风格。
- 核心功能稳定优先。
- Provider 管理、配置备份、MCP 管理等功能不能依赖注入。
- Codex++ 注入功能必须隔离：
  - 可开启 / 可关闭
  - 有版本检测
  - 有日志和诊断
  - 失败不影响主程序
- 不明文展示 token、Cookie、auth.json、密钥。
- 不直接修改真实配置文件，除非先完成备份和 dry-run diff。
- 第一版不做 Gemini / OpenCode / 复杂云同步 / 多语言完整 i18n。
